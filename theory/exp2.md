# Theory
Graph isomorphism is a concept in graph theory used to determine whether two graphs are structurally identical. Two graphs are said to be isomorphic if there exists a one-to-one correspondence between their vertices such that the adjacency relationships are preserved. In simple terms, even if two graphs look different when drawn or have different vertex labels, they may still represent the same structure if the pattern of connections between the vertices is the same. Therefore, isomorphism focuses on structural equivalence rather than visual similarity.
To verify whether two graphs are isomorphic, several structural properties of the graphs can be compared. These properties are often called graph invariants because they remain unchanged under isomorphism. Some common invariants include the number of vertices, number of edges, degree sequence, shortest path lengths between nodes, and the number of cycles of various lengths. If any of these properties differ between two graphs, then the graphs cannot be isomorphic.
In this experiment, two graphs are constructed using the NetworkX library in Python. The graphs are created by first defining an empty graph using nx.Graph() and then adding edges using the function add_edges_from(). Each graph contains ten vertices connected in a specific pattern. The two graphs appear visually different, but the program checks whether their structural properties are equivalent.
The program first analyzes the distribution of shortest path lengths in each graph. The shortest path length between two vertices represents the minimum number of edges that must be traversed to travel from one vertex to the other. In the code, this is calculated using the function nx.single_source_shortest_path_length(). This function finds the shortest distance from one vertex to all other vertices in the graph. The results are stored using the Counter class from the Python collections module, which counts how many node pairs occur at each distance. The resulting dictionary shows how many paths of length 0, 1, 2, and so on exist in the graph. If the distribution of shortest path lengths is different between two graphs, it indicates that the structures are different.
Another important structural feature examined in the program is the number of cycles of different lengths. A cycle in a graph is a closed path where the starting and ending vertex are the same and no edges are repeated. Cycles help describe how loops are formed within a graph. In the code, the function kcycles() is used to count cycles of specific sizes. The program checks combinations of vertices using the combinations() function from the itertools module. For each selected set of vertices, the code creates a subgraph using G.subgraph() and then verifies whether the subgraph is connected using nx.is_connected() and whether it has the required number of edges. In this experiment, the number of 4-cycles and 5-cycles is calculated for both graphs. If the cycle counts differ, the graphs cannot be isomorphic.
After comparing these structural properties, the program performs a formal isomorphism test. This is done using the NetworkX class GraphMatcher. The object nx.isomorphism.GraphMatcher(G1, G2) attempts to find a bijective mapping between the vertices of the two graphs. The method is_isomorphic() checks whether such a mapping exists. If the method returns true, the graphs are considered isomorphic and the corresponding vertex mapping is stored. If no mapping satisfies the adjacency condition, the graphs are declared non-isomorphic.
Visualization of the graphs is performed using the Matplotlib library together with NetworkX drawing functions. The function nx.draw() is used to display the graphs by plotting vertices and edges based on predefined coordinate positions. The plt.figure() function creates a figure window, while fig.add_gridspec() and add_subplot() are used to organize the layout so that both graphs and the comparison information can be displayed in a structured manner. Titles and text panels are added to show the comparison results such as shortest path length counts, cycle counts, and the isomorphism mapping.
By combining structural analysis with algorithmic isomorphism checking, the program verifies whether two graphs represent the same connectivity pattern. This experiment demonstrates how graph properties such as shortest path lengths and cycle counts can be used to compare graph structures before applying a formal isomorphism algorithm. Through visualization and computational verification, the experiment provides a clear understanding of how graph isomorphism works in practice
