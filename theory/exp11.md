# Experiment 11

# Aim
To implement Greedy graph coloring algorithm that assigns colors to the vertices such that no 2 adjacent vertices share same color while minimizing the chromatic number.

# Theory
Graph colouring is a method in graph theory used to assign colours to the vertices of a graph in such a way that no two adjacent vertices have the same colour. The main objective of graph colouring is to use the minimum number of colours while satisfying the colouring condition. This minimum number of colours required to colour a graph is called the chromatic number of the graph. Graph colouring has many practical applications such as map colouring, scheduling problems, register allocation in compilers, frequency assignment in mobile networks and timetabling problems. In these applications, the main purpose is to avoid conflicts between connected or related elements.
In this experiment, a greedy graph colouring algorithm is implemented. The greedy method works by selecting vertices one by one and assigning them colours according to certain rules. It always chooses the smallest available colour that has not already been assigned to any adjacent vertex. Although the greedy method does not always guarantee the absolute minimum number of colours for every graph, it is simple, fast and gives efficient results for many practical situations.
For this experiment, the DSATUR (Degree of Saturation) strategy is used with the greedy colouring algorithm. The DSATUR method selects the next vertex based on its saturation degree. Saturation degree means the number of different colours already assigned to its neighbouring vertices. The vertex having the highest saturation degree is selected first. If two vertices have the same saturation degree, then the vertex having the larger degree (larger number of connected edges) is selected. This strategy generally produces better colouring results and often reduces the number of colours used.
In the implementation, a graph is first created by adding vertices and edges according to the given graph structure. Since the graph contains multiple edges between some vertices, a MultiGraph structure is used. NetworkX provides built-in functions that simplify graph operations. The add_edges_from() function is used to add edges to the graph. The greedy_color() function is used to perform graph colouring and the strategy parameter "saturation_largest_first" is used to apply the DSATUR method. The colours returned by the algorithm are mapped to RGB colours such as red, green, blue and yellow for easier visualization.
For displaying the graph, NetworkX drawing functions are used. The draw_networkx_nodes() function is used to draw vertices, draw_networkx_labels() is used to display vertex numbers and draw_networkx_edges() is used to draw edges between vertices. Parallel edges between some vertices are drawn separately using curved connections so that both edges are visible clearly. Finally, the graph is displayed along with the assigned colours and the chromatic number is calculated and printed.
The greedy colouring algorithm with DSATUR strategy improves the colouring process by selecting vertices in an efficient order. By assigning colours systematically and checking neighbouring vertices at every step, the algorithm successfully colours the graph without violating graph colouring rules. This method helps in reducing conflicts and minimizing the number of colours required.

# Conclusion
A greedy graph colouring algorithm was successfully implemented to assign colours to graph vertices such that no two adjacent vertices had the same colour while using the minimum possible colours.
