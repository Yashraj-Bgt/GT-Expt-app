# Experiment 03

# Aim
To implement and generate different types of subgraphs such as spanning subgraph, vertex-induced subgraph, and edge-induced subgraph from a given graph.

# Theory
In graph theory, a subgraph is a graph that is formed from a subset of the vertices and edges of another graph. Subgraphs help in analyzing smaller structures within a larger network and are widely used in many applications such as network analysis, circuit design, and social network modeling. By studying subgraphs, it becomes easier to understand the relationships and connectivity patterns present in complex graphs.
In this experiment, a base graph is first created containing a set of vertices labeled from A to I and a number of edges connecting them. This original graph represents the main structure from which different types of subgraphs are generated. The graph consists of vertices and edges that form a connected network, and its layout is defined using fixed coordinate positions so that the graph can be displayed clearly using Matplotlib.
One important type of subgraph is the spanning subgraph. A spanning subgraph contains all the vertices of the original graph but only a subset of its edges. In other words, no vertices are removed, but some connections between them may be omitted. In the program, the spanning subgraph is generated by selecting specific edges from the original edge list while keeping all the vertices. This helps demonstrate how the structure of the graph can be simplified while still including every vertex. Another type of subgraph generated in the program is the induced subgraph, also known as a vertex-induced subgraph. In this case, a subset of vertices is selected from the original graph. Once these vertices are chosen, all edges that exist between those vertices in the original graph are automatically included in the induced subgraph. In the code, a list of selected vertices is created, and then the program checks the original edge list to determine which edges connect those vertices. Only those edges are included when drawing the induced subgraph.
The experiment also demonstrates an edge-induced subgraph. In this type of subgraph, a subset of edges from the original graph is selected. The vertices associated with those edges are included automatically. Unlike vertex-induced subgraphs, the selection here is based on edges rather than vertices. The code defines a list of chosen edges, and the vertices that appear in those edges are collected and used to draw the edge-induced subgraph.
The visualization of these graphs is performed using the Matplotlib library. Matplotlib provides tools to create figures and display graphical information. The function plt.subplots() is used to create a grid of plotting areas so that multiple graphs can be shown in the same window. Each subplot represents a different graph: the original graph, spanning subgraph, induced subgraph, and edge-induced subgraph.
Edges are drawn by plotting lines between the coordinates of two connected vertices, while vertices are drawn using scatter points. The function ax.plot() is used to draw edges and ax.scatter() is used to represent nodes. The node labels are displayed using ax.text(), which places the vertex name at the corresponding coordinate. Titles are added using set_title() to identify each graph clearly. Finally, plt.tight_layout() organizes the layout to avoid overlapping elements, and plt.show() displays the complete figure.
By implementing these different types of subgraphs, the experiment demonstrates how smaller graphs can be derived from a larger graph while preserving specific structural relationships.

# Conclusion
Generation and visualization of spanning, induced, and edge-induced subgraphs from a given graph was successfully implemented.
