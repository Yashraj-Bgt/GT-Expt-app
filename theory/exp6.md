# Theory
A graph is a mathematical structure used to represent relationships between objects. It is denoted as , where represents the set of vertices (nodes) and represents the set of edges connecting those vertices. In many real-life situations like road networks, communication systems, or wiring connections, graphs are used along with weights on edges. These weights can represent distance, cost, or time. Such graphs are called weighted graphs.
One important problem in graph theory is connecting all the vertices of a graph with minimum total cost. This leads to the concept of a Minimum Spanning Tree (MST). A spanning tree is a subgraph that includes all the vertices of the original graph, remains connected, and does not contain any cycles. For a graph with vertices, a spanning tree will always have exactly edges. Among all possible spanning trees, the one with the smallest total edge weight is called the Minimum Spanning Tree.
To find the MST, one of the most commonly used methods is Kruskal’s Algorithm. It is a greedy algorithm, which means at every step it selects the edge with the minimum weight. The algorithm starts by sorting all the edges of the graph in increasing order of their weights. Then it begins adding edges one by one. While adding edges, it checks whether adding that edge will create a cycle. If a cycle is formed, that edge is skipped. This process continues until all vertices are connected using edges.
An important concept in Kruskal’s algorithm is the idea of components. Initially, each vertex is considered as a separate component. As edges are added, these components start merging. To efficiently check whether two vertices belong to the same component, a method called Union-Find (Disjoint Set) is used. The “find” operation identifies the component of a vertex, and the “union” operation merges two components. If two vertices already belong to the same component, adding an edge between them will form a cycle, so that edge is not included.
In this experiment, we have implemented Kruskal’s algorithm using Python. The graph is taken as input in the form of a weighted adjacency matrix, where each value represents the weight between two vertices. From this matrix, all edges are extracted and stored along with their weights. These edges are then sorted in increasing order using Python’s sorting function, which follows the basic idea of Kruskal’s algorithm.
In our code, instead of directly using a built-in MST function, we have implemented the logic manually using union-find operations. A parent structure is maintained to keep track of components. The “find” function is used to determine the parent (component) of a vertex, and the “union” function is used to merge components when a valid edge is added. This helps in preventing cycles during the MST construction.
During execution, edges are added step by step only if they do not form a cycle. After each step, the current set of selected edges is stored. Using Matplotlib, the graph is displayed for every step. All edges of the graph are shown in light color, and the edges selected in the MST are highlighted clearly. This step-by-step visualization helps in understanding how the MST is formed gradually.
Thus, this experiment helps in understanding both the theoretical concept of Minimum Spanning Tree and the working of Kruskal’s algorithm. It also shows how components are merged and how cycles are avoided while constructing the MST in a clear and visual manner.
