# Experiment 05

# Aim
Convert the original graph into its line graph, where each edge of the original graph becomes a vertex in the new graph, and adjacency is defined by shared endpoints in the original graph.

# Theory
In graph theory, a line graph is a special type of graph that is constructed from another graph. Instead of focusing on vertices like in normal graphs, a line graph focuses on edges. In a line graph, every edge of the original graph becomes a vertex in the new graph. Then, two vertices in the line graph are connected if the corresponding edges in the original graph share a common endpoint. So basically, we are converting relationships between vertices into relationships between edges.
In this experiment, the original graph is given in the form of an adjacency matrix. An adjacency matrix is a square matrix where each row and column represents a vertex. If there is an edge between two vertices, the corresponding position in the matrix is marked as 1, otherwise it is 0. Using this matrix, we first identify all the edges present in the graph by checking which positions contain the value 1. This step is important because these edges will later become the vertices of the line graph.
Once the edges are identified, the next step is to construct the line graph. According to the definition, each edge of the original graph becomes a node in the line graph. After that, we check every pair of edges in the original graph. If two edges share a common vertex, then we connect their corresponding nodes in the line graph. This step ensures that adjacency in the line graph correctly represents shared endpoints from the original graph.
In the first part of the program, the construction is done using the NetworkX library. The function nx.from_numpy_array() is used to convert the adjacency matrix into a graph. After creating the graph, the function nx.line_graph() is used to directly generate the line graph. This function automatically performs all the steps internally, such as identifying edges, converting them into vertices, and connecting them based on shared endpoints. The resulting graph is then displayed using the nx.draw() function. This makes the implementation simple and quick.
In the second part of the program, the same process is implemented manually without using NetworkX functions. Here, we explicitly extract edges from the adjacency matrix using loops. Then, we create a list of edges and treat each edge as a node in the new graph. After that, we compare each pair of edges. If they have a common vertex, we add an edge between them in the line graph. This approach helps in understanding how the line graph is actually formed step by step, instead of relying on built-in functions.
For visualization, Matplotlib is used in both cases. The function plt.figure() creates the plotting window, and plt.subplot() is used to display both the original graph and the line graph side by side. In the manual implementation, edges are drawn using plt.plot() and nodes are drawn using plt.scatter(). Labels are added using plt.text() so that each vertex or edge can be clearly identified. The positions of nodes are arranged in a circular pattern using trigonometric functions, which makes the graph easier to understand visually.
By performing this experiment, it becomes clear how a graph can be transformed into its line graph and how the focus shifts from vertices to edges. It also shows the importance of understanding both the theoretical concept and the practical implementation. Using NetworkX makes the process easier, but implementing it manually gives a deeper understanding of how adjacency relationships are formed in a line graph.
