# Experiment 04

# Aim
To determine and construct a graph corresponding to a given degree sequence and verify whether the sequence is graphical.

# Theory
In graph theory, the degree of a vertex represents the number of edges connected to that vertex. A degree sequence is a list that contains the degrees of all vertices of a graph arranged in any order. Determining whether a valid graph can be formed from a given degree sequence is an important problem in graph theory. A degree sequence is said to be graphical if there exists at least one simple graph whose vertices have exactly the degrees specified in the sequence.
One commonly used approach to determine whether a sequence is graphical is the Havel–Hakimi algorithm. This algorithm works by repeatedly selecting the vertex with the highest degree and connecting it to other vertices with the next highest degrees. After each connection, the degree values of the corresponding vertices are reduced. The process continues until either all degrees become zero, indicating that a valid graph can be constructed, or a negative value appears, indicating that the sequence cannot represent a valid graph.
In this experiment, the user first provides a degree sequence as input. The program then checks whether the sequence is graphical and constructs the corresponding graph if possible. Two different approaches are used in the experiment: one using the NetworkX library and another using manual implementation.
In the NetworkX implementation, the function nx.is_graphical() is used to verify whether the given degree sequence can represent a valid graph. If the sequence is graphical, the function nx.havel_hakimi_graph() automatically constructs a graph that satisfies the degree sequence using the Havel–Hakimi algorithm. This function internally performs the reduction process and creates the required edges between vertices. The graph is then visualized using the nx.draw() function, which displays the nodes and edges. The node positions are generated using the nx.spring_layout() function, which arranges vertices in a visually balanced layout.
The second program performs the same task without using NetworkX functions. In this version, the Havel–Hakimi algorithm is implemented manually. The program pairs each vertex with its degree and sorts the list in descending order. The vertex with the highest degree is selected and connected to the next set of vertices according to its degree value. After creating each connection, the degrees of the connected vertices are reduced. If any degree becomes negative during the process, the sequence is declared non-graphical. If the process finishes successfully and all degrees become zero, a valid graph has been constructed.
For visualization in the manual implementation, Matplotlib is used directly. The vertices are plotted using plt.scatter() and labeled using plt.text(). The edges between vertices are drawn using plt.plot(). The function plt.figure() creates the plotting area, while plt.show() displays the final graph. This allows the structure of the graph generated from the degree sequence to be visualized clearly.
Through this experiment, the concept of graphical sequences and the Havel–Hakimi algorithm is demonstrated. The experiment shows how a degree sequence can be analyzed and used to construct a graph structure both automatically using NetworkX and manually using basic Python logic.

# Conclusion
Construction of a graph corresponding to a given degree sequence using the Havel–Hakimi method was successfully implemented.
